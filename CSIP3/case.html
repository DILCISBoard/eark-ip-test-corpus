Rules

  • Rule:1 Packages:0/2
    If mets/@TYPE = "OTHER" then mets/@csip:OTHERTYPE MUST NOT have a value from the VocabularyContentCategory because these vocabulary values MUST be used for the attribute mets/@TYPE.

    ERROR: mets/@csip:OTHERTYPE attribute has a value that MUST instead be used for mets/@TYPE. MUST be: mets/@csip:OTHERTYPE MUST describe a content category that falls outside of the VocabularyContentCategory vocabulary.

  • Rule:2 Packages:0/2

    Packages: mets-xml_mets_TYPE_not_OTHER_and_csip-OTHERTYPE_exists  minimal_IP_with_1_rep_mets_TYPE_OTHER
